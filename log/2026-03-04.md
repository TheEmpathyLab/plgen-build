## What was built

### Registry server (from scratch)
- `server.js` — Express app with helmet, cors, rate limiting
- `src/db.js` — SQLite via better-sqlite3, WAL mode, auto-creates db/
- `src/auth.js` — API key auth via `x-plgen-key` header
- `src/routes/labels.js` — register, fetch, list labels
- `src/routes/members.js` — admin signup, member profile
- `src/routes/viewer.js` — public HTML viewer pages at `/:plId`

### Deployment
- DigitalOcean Ubuntu 22.04 droplet at `67.205.169.17`
- Node.js 20, nginx, pm2
- SSL via certbot — `registry.provenancelabel.org`
- DNS A record added in Namecheap
- GitHub private repo with deploy key for pull-based deploys
- Daily SQLite backup cron via `crontab`

### Stripe + Resend integration
- `POST /webhooks/stripe` — handles `checkout.session.completed`
- Auto-creates member, generates `plg_live_` API key
- Sends welcome email via Resend with API key

---

## Decisions made

- **SQLite over Postgres** — sufficient for current scale, zero ops overhead
- **API key auth** — simple for MVP, designed to be replaced by proper auth later
- **Admin-only member creation** — Stripe handles payment, webhook calls admin endpoint
- **Two tiers at launch** — free (text label only) and registered (permanent URL + badge)

---

## Issues encountered

- `.env` heredoc wouldn't write correctly via SSH terminal — used `nano` instead
- `express.json()` was parsing webhook body before Stripe signature check — fixed by mounting webhook route before `express.json()`
- `pm2 save` command not found when chained — ran separately
- nginx config file writing via terminal wrapping — solved with `nano`
