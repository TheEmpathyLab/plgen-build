# Session Log — 2026-04-02

**Focus:** Magic link auth, dashboard registration panel, ChatGPT Custom GPT v3.x, registry GitHub workflow

## What was decided

- **API key auth is out.** Replaced entirely with magic link + HMAC session tokens. No credentials to manage, lose, or paste.
- **Registration is members-only.** Guest/anonymous label registration removed — too much storage overhead and financial strain at scale. Only paying members can register labels with a permanent URL.
- **Registration lives in the dashboard, not a public page.** The public `/register` page is now obsolete. Members paste JSON from PLGen into the dashboard panel and register from there.
- **JSON is the one format.** PLGT1 token format (pipe-delimited) retired. JSON is what the GPT outputs, what the dashboard accepts, and what the Actions API will use. One direction, no both.
- **ChatGPT Custom GPT is the priority AI platform.** ~55-65% US market share vs. Claude at ~5-10%. GPT Actions can make authenticated API calls today — Claude requires MCP (longer term).
- **In-conversation registration architecture:**
  - **Now (Claude):** `plgen generate` → copy JSON → paste into dashboard. One web visit, minimal friction.
  - **Now (GPT, no Actions yet):** Same — generate → copy → dashboard.
  - **Near future (GPT with Actions):** Full in-conversation `plgen register` fires API call directly. Issue #3.
  - **Long term (Claude):** MCP server. Issue #4.
- **`work_title` added to JSON format.** Was missing — GPT prompt updated to include it so the dashboard registration panel can auto-populate it.
- **Label editing scoped narrowly.** Only `work_url` and `work_title` are editable post-registration. Contribution split, author, and process notes are immutable — they represent the original disclosure.

## What was built

### Registry server (`provenancelabel/registry`)

- **GitHub workflow established** — repo was on droplet only, now cloned locally, all changes via git push + pull on droplet. Deploy = `git pull && pm2 restart plgen-registry`.
- **`src/auth.js`** — added `generateSessionToken(memberId, email)` and `validateSessionToken(token)`. HMAC-SHA256 signed, 30-day expiry, base64url encoded. `authMember` kept for backwards compat.
- **`src/db.js`** — added safe migrations for `login_token` and `login_token_expires` columns on members table.
- **`src/routes/members.js`** — added:
  - `POST /api/members/request-login` — validates email, generates 32-byte hex token, stores with 15-min expiry, sends magic link via Resend. Always returns success (prevents email enumeration).
  - `GET /api/members/verify-login?token=XXX` — validates token + expiry, consumes it, generates session token, redirects to `/dashboard?session=TOKEN`.
- **`src/routes/dashboard.js`** — major updates:
  - Auth: email input → "check inbox" state → magic link → session token (replaces API key input entirely)
  - `?token=` query param on `/dashboard` redirects to verify-login
  - `?session=` query param stores to sessionStorage, strips from URL bar
  - `POST /dashboard/register` endpoint — session-authenticated, inserts label as `registered` tier, returns `pl_id` + permanent URL
  - Registration panel UI: JSON paste → Parse JSON → preview (author, split, tools, process) → Register Label → success with permanent URL
  - Label history refreshes after successful registration

### ChatGPT Custom GPT (`provenancelabel/provenancelabel/custom-gpt/`)

- **`custom-gpt/` folder created** — dedicated home for GPT instruction builds
- **`system-prompt-v2.md`** — archived (original, no registration)

  - v3.1: `plgen register` now members-only, directs to `/dashboard` not public form. Guest path removed.
  - v3.2: added `work_title` to JSON output format

### Register page (`provenancelabel/provenancelabel/register/`)
