## Decisions made

- Delta is computed server-side by parsing `_token_raw` directly — assumed PLGT1 pipe-delimited format: `PLGT1|author|human_pct|ai_pct|ai_tools|process_notes|session_init|estimated|checksum`
- If token parsing fails (malformed raw), delta stored as null — graceful degradation
- `_flagged` field only appears in response when token was used AND checksum was invalid — clean, non-breaking addition to response shape
- member_id attribution reviewed — was already correct in existing code (`member?.id || null`); made explicit with named variable for clarity
